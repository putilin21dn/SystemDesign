# Фитнес-трекер - архитектура системы

**Вариант 14**

Система предназначена для хранения информации о тренировках
пользователей, упражнениях и статистике тренировок.\
Аналог сервиса: https://www.myfitnesspal.com/

------------------------------------------------------------------------

# 1. Описание системы

Фитнес-трекер - это приложение, которое позволяет пользователям
отслеживать свою физическую активность и вести историю тренировок.

Основные возможности системы:

-   регистрация пользователей
-   поиск пользователей
-   создание упражнений
-   создание тренировок
-   добавление упражнений в тренировку
-   просмотр истории тренировок
-   получение статистики тренировок за период

------------------------------------------------------------------------

# 2. Основные сущности системы

## Пользователь (User)

Хранит информацию о пользователе приложения.

Поля:

-   id
-   login
-   name
-   surname
-   email
-   created_at

------------------------------------------------------------------------

## Упражнение (Exercise)

Справочник упражнений.

Поля:

-   id
-   name
-   description
-   muscle_group
-   created_at

------------------------------------------------------------------------

## Тренировка (Workout)

Хранит информацию о тренировке пользователя.

Поля:

-   id
-   user_id
-   date
-   duration
-   created_at

------------------------------------------------------------------------

## Упражнение в тренировке (WorkoutExercise)

Связующая сущность между тренировкой и упражнениями.

Поля:

-   id
-   workout_id
-   exercise_id
-   sets
-   reps
-   weight

------------------------------------------------------------------------

# 3. API системы

## Создание нового пользователя

POST /users

Request:

{ "login": "ivan123", "name": "Ivan", "surname": "Ivanov", "email":
"ivan@mail.com" }

Response:

{ "id": 1, "login": "ivan123" }

------------------------------------------------------------------------

## Поиск пользователя по логину

GET /users/{login}

------------------------------------------------------------------------

## Поиск пользователя по имени и фамилии

GET /users/search?name=ivan

------------------------------------------------------------------------

## Создание упражнения

POST /exercises

Request:

{ "name": "Bench Press", "description": "Chest exercise",
"muscle_group": "chest" }

------------------------------------------------------------------------

## Получение списка упражнений

GET /exercises

------------------------------------------------------------------------

## Создание тренировки

POST /workouts

Request:

{ "user_id": 1, "date": "2026-03-11" }

------------------------------------------------------------------------

## Добавление упражнения в тренировку

POST /workouts/{workout_id}/exercises

Request:

{ "exercise_id": 3, "sets": 4, "reps": 10, "weight": 80 }

------------------------------------------------------------------------

## Получение истории тренировок пользователя

GET /users/{id}/workouts

------------------------------------------------------------------------

## Получение статистики тренировок за период

GET /users/{id}/stats?from=2026-01-01&to=2026-03-01

Возвращает:

-   количество тренировок
-   суммарный поднятый вес
-   длительность тренировок

------------------------------------------------------------------------

# 4. Архитектура системы

Система построена по клиент-серверной архитектуре.

Основные компоненты:

1.  Веб 
2.  Backend API
3.  База данных
4.  Сервис аналитики

------------------------------------------------------------------------

# 5. Контейнеры системы

## Веб 

Назначение:

-   пользовательский интерфейс
-   отправка запросов к backend API

Технологии:

-   React
-   Mobile App

------------------------------------------------------------------------

## Backend API

Назначение:

-   обработка REST запросов
-   реализация бизнес-логики
-   взаимодействие с базой данных

Технологии:

-   Node.js

------------------------------------------------------------------------

## База данных

Назначение:

-   хранение пользователей
-   хранение тренировок
-   хранение упражнений

Технологии:

-   PostgreSQL

------------------------------------------------------------------------

## Сервис аналитики

Назначение:

-   расчет статистики тренировок
-   анализ активности пользователей

Технологии:

-   Python

------------------------------------------------------------------------

# 6. Взаимодействие компонентов

Пользователь взаимодействует с системой через веб или мобильный клиент.

Поток работы:

1.  Пользователь отправляет запрос через клиентское приложение
2.  Клиент отправляет REST запрос в Backend API
3.  Backend обрабатывает запрос
4.  Backend взаимодействует с базой данных
5.  Backend возвращает результат клиенту

------------------------------------------------------------------------

# 7. Основной сценарий работы

### Создание тренировки

1.  Пользователь нажимает кнопку "Создать тренировку"
2.  Клиент отправляет запрос:

POST /workouts

3.  Backend API проверяет данные пользователя
4.  Backend сохраняет тренировку в базе данных
5.  Backend возвращает id тренировки
6.  Пользователь может добавить упражнения в тренировку
